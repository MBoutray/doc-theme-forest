Installation

  1. Unzip the template folders

    Unzip the archive named template that holds the files of the template.

  2. Customize the content of the template

    Using the information given in this documentation, modify the content as you see fit (text, images, colors...).
    You can preview the changes that you make by opening the index.html file in a browser and reloading the page after the changes.

  3. Upload the template

    Use an FTP client to connect remotely to your hosting server and upload the files to the root of the server.

  4. Admire your new website !

    You're done! Your new website should be visible on the internet.

HTML

All the content of the website is held in the index.html file at the root of the template.

The website was coded using TailwindCSS and SASS so you'll find a LOT of classes in the HTML, but you don't need to worry about it to modify the content. (If you have experience in coding for the web, please be understanding of the way I coded as it was a school assignment, it might not be perfect :) )

HTML structure

The header of the page

The header contains the logo and the navigation links.
You can modify the path to the logo and your portrait for the images.

If you don't want a section on the page, you can delete it and adapt the menus by removing their link (don't forget that there is two menus, one for the mobile version and one for the desktop version).

CSS

The CSS was create using Tailwind and SASS and then combined using the gulp-postcss plugin for Gulp.

You can modify the generated style.css file manually or you could use the dev-dependencies to compile the styles code.

I added some comments in the stylesheets that start by CUSTOMIZE where it is important/interesting for you to modify the content of the code (for the hero image for example)

#hero-section {
  /* CUSTOMIZE: here modify the name of the background image for the hero section */
  background: url("../images/jpg/hero-mobile.jpg") no-repeat center;
  [...]
}

Images

In general, website use <img> tags for displaying images but they have the inconvenience that the images are not easily swappable for mobile versions of the site.

Swapping images for the mobile and desktop version is very important to have a website that is fast to load on mobile as mobile connections are slower to load big image files.

I used the <picture> tag and <source> tags to allow seamless swapping of the images.

<picture>
  <!-- Desktop version -->
  <source media="(min-width: 1024px)" 
          srcset="./path/to/desktop-image.webp"
          type="image/webp">
  <source media="(min-width: 1024px)" 
          srcset="./path/to/desktop-image.jpg"
          type="image/jpg">

  <!-- Mobile version -->
  <source media="(max-width: 1024px)" 
          srcset="./path/to/mobile-image.webp"
          type="image/webp">
  <source media="(max-width: 1024px)" 
          srcset="./path/to/mobile-image.jpg"
          type="image/jpg">

  <!-- Default image if no <source> is supported -->
  <img src="./path/to/default-image.jpg" alt="Image description">
</picture>

I also used webp images for the website; webp images are next-generation images that have a way better compression than .jpg or .png images, so images are way lighter and better quality. It is advised to use them for better page load times.

However most likely your images are .jpg and .png, so you'll need to convert them using a file converter or image editor (like Photoshop). You may also don't want to bother and then you can just replace the <picture> tag with an <img src="path/to/file.extension" alt="Image description"> tag to your image.

Advanced way to generate webp images

Using the node package manager (npm), you can download the dev-dependencies using npm install.
Put all the images you need to convert into webp in the images/jpg and images/png folders and execute the npm run webp command at the root of the folder.
